# Calvin

**所属团队**: Vida
**背景**: 清华大学 00 后开发者
**研究方向**: Proactive Agent（主动式 Agent）

## 概述

Calvin 是 [[openchronicle]] 项目负责人及 Vida 团队核心成员，主要研究方向为 Proactive Agent——能根据用户上下文主动判断、主动建议、主动行动的 Agent，区别于当前主流的 reactive Agent（不问不动）。^[raw/articles/2026-04-27_记忆，是Agent基建｜对话Calvin@Vida.md]

## 核心观点

### 记忆是 Agent 基建

Calvin 认为，随着模型能力趋于均衡（OpenAI、Claude、国内开源差距缩小），真正影响用户体验的是模型拥有的关于用户的记忆与上下文多少。记忆正从"某个产品的差异化功能"变成"Agent 时代的基础设施"。^[raw/articles/2026-04-27_记忆，是Agent基建｜对话Calvin@Vida.md]

### 记忆归属用户

Agent 记忆的所有权应归属用户而非模型厂商。记忆该是设备里的一层基础设施，跟谁的模型协作都行。各家大模型厂商争夺用户上下文以提高迁移成本的做法，本质上是将用户变为厂商资产。^[raw/articles/2026-04-27_记忆，是Agent基建｜对话Calvin@Vida.md]

### 开源策略：掀桌子

面对记忆赛道的融资热潮，Calvin 选择开源 [[openchronicle]]，策略是"掀桌子然后看谁能在场景里给出 Best Practice"。该团队从 2025 年底开始做记忆系统，在 OpenAI Chronicle 发布时已积累深厚技术储备。^[raw/articles/2026-04-27_记忆，是Agent基建｜对话Calvin@Vida.md]

## 与 [[self-improving-agent]] 的关联

Calvin 对 Proactive Agent 的研究——让 Agent 从 reactive 进化到 proactive——与 [[self-improving-agent]] 的设计目标高度契合。OpenChronicle 提供的设备侧记忆基础设施正是 Proactive Agent 的关键支撑：只有"知道你怎么做事"，Agent 才能主动行动而非被动响应。
