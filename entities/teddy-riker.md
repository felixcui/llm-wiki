# Teddy Riker

**身份**: Ramp 产品团队成员
**领域**: AI Agent 产品设计、企业 SaaS

## 概述

Teddy Riker 是企业支出管理平台 Ramp 的产品团队成员，以原创文章 "Designing for Agents" 系统阐述了为 AI 智能体设计产品的三大核心原则。该文由宝玉翻译为中文后在微信公众号广泛传播。^[raw/articles/2026-04-26-designing-products-for-agents.md]

## 核心观点

### 为 Agent 设计产品三原则

1. **教会 AI 智能体如何成功**：主动把调用方智能体需要知道的信息交给它，而不是让它自己摸索。例如 Notion 的 MCP 工具会在智能体写入前先提供增强 Markdown 规范。
2. **建立反馈循环**：通过 rationale 参数、反馈工具和上下文种子，从智能体交互中收集产品改进信号。
3. **留意上下文缺口**：在智能体到智能体的交互中，清楚判断哪一方在哪些信息上更有优势，设计好上下文交换机制。^[raw/articles/2026-04-26-designing-products-for-agents.md]

### 软件交互范式转变

提出软件交互正在从"用户 → 界面 → 数据库"向"用户 → 用户智能体 → 软件智能体 → 数据库"演进，认为 UI 不会消失但二八法则已经反过来——未来 80% 的交互将通过 AI 智能体完成。^[raw/articles/2026-04-26-designing-products-for-agents.md]
