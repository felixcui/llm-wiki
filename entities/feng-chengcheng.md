# 冯成成

**类型**: 人物
**公司**: 阿里云
**职位**: 阿里云专有云 Agent 技术开发负责人

## 概述

冯成成是阿里云专有云 Agent 技术开发负责人，深度参与 [[openclaw]] 及企业级 Agent 落地的研究与开发。在专有云 Opentrek 智能体平台中负责 Agent 企业级落地的安全管控、权限体系、协同机制以及生产部署。^[raw/articles/2026-04-29_深度访谈｜OpenClaw引爆Agent元年，AIAgent在企业内如何规模化应用？.md]

## 核心观点

### OpenClaw 爆火的本质

冯成成认为 OpenClaw 代表 AI 交互范式的质变——从"对话式 AI"推进到"执行式 AI"。爆火前提包括底层模型能力够强、使用门槛低。^[raw/articles/2026-04-29_深度访谈｜OpenClaw引爆Agent元年，AIAgent在企业内如何规模化应用？.md]

### 用户从"试试"到"用起来"的三道墙

1. **场景墙**：用户不知让 Agent 做什么，需要从"问问题"转向"交任务"
2. **成本墙**：Token 消耗带来的经济压力，需选择高价值场景
3. **信任墙**：担心 Agent 误操作、隐私泄露，最难跨越^[raw/articles/2026-04-29_深度访谈｜OpenClaw引爆Agent元年，AIAgent在企业内如何规模化应用？.md]

### Agent 三维评估框架

提出企业级 Agent 应从三个维度评估：**可控**（全生命周期审计、权限分级）、**有能力**（复杂指令处理、工具链、记忆能力）、**能协同**（多 Agent 交互、标准协议）。^[raw/articles/2026-04-29_深度访谈｜OpenClaw引爆Agent元年，AIAgent在企业内如何规模化应用？.md]

### OpenClaw vs HermesAgent 架构差异

- OpenClaw：**"人教 Agent 做事"** — 简单、可控、易审计，但缺乏自我进化能力
- [[hermes-agent]]：**"Agent 自己学会做事"** — 完整记忆框架，Skill 可在执行中沉淀
- 长期来看，成熟框架应同时具备稳定可控的执行底座和持续学习能力^[raw/articles/2026-04-29_深度访谈｜OpenClaw引爆Agent元年，AIAgent在企业内如何规模化应用？.md]

### Agent 落地的四步指南

1. 选对场景，从简单、容错率高的场景切入
2. 建立稳定安全的基础能力（权限分级、身份认证、审计、沙箱）
3. 建设丰富的企业级能力（业务数据、流程、经验标准化）
4. 建立可观测、自救和自进化能力^[raw/articles/2026-04-29_深度访谈｜OpenClaw引爆Agent元年，AIAgent在企业内如何规模化应用？.md]

### 对 Agent 未来的判断

Agent 已证明自己能干活。未来当它真正做到"管得住、有深度、能协同"时，会像云计算一样成为企业标配。^[raw/articles/2026-04-29_深度访谈｜OpenClaw引爆Agent元年，AIAgent在企业内如何规模化应用？.md]
