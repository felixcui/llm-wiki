# 姚柳佚

**类型**: 人物
**公司**: 阿里巴巴
**职位**: 阿里巴巴 ATH 事业群通义大模型事业部科学家，[[qwenpaw]] 产品技术负责人之一

## 概述

姚柳佚是阿里巴巴 ATH 事业群通义大模型事业部科学家，也是 [[qwenpaw]]（前身 coPaw）产品技术的负责人之一，专注于 Agent 产品体验、开源生态与用户路径。^[raw/articles/2026-04-29_深度访谈｜OpenClaw引爆Agent元年，AIAgent在企业内如何规模化应用？.md]

## 核心观点

### OpenClaw 爆火三要素

姚柳佚认为 OpenClaw 爆火源于三个因素叠加：**模型能力到了、产品体验顺了、开源传播快了**。尤其是产品层面——它不再只是远程问答工具，而是真正"长"在本地环境里，支持 IM 渠道、心跳机制、定时任务。^[raw/articles/2026-04-29_深度访谈｜OpenClaw引爆Agent元年，AIAgent在企业内如何规模化应用？.md]

### 从"试试"到"用起来"的两件事

1. **场景适配**：必须与日常高频场景结合，否则只是"虚火"
2. **信任感**：建立对 Agent 执行能力和安全边界的信任^[raw/articles/2026-04-29_深度访谈｜OpenClaw引爆Agent元年，AIAgent在企业内如何规模化应用？.md]

### OpenClaw vs HermesAgent

- OpenClaw：插件系统和 Hook 机制，适合需要确定性和安全边界的场景
- [[hermes-agent]]：Learningloop（学习闭环），潜力更大，能力天花板更高，但消耗更多 Token
- 两者适合不同需求，非替代关系^[raw/articles/2026-04-29_深度访谈｜OpenClaw引爆Agent元年，AIAgent在企业内如何规模化应用？.md]

### 企业级 Agent 落地的三步走

1. **小范围试验田**：验证能否跑通
2. **大范围验证**：扩展场景和权限边界
3. **工厂化落地**：纳入生产体系
- 每一步都要沉淀安全规则和安全评判基准
- Human in the Loop 应分级，非完全自主也非事事审批
- 未来可能出现 **AgentOps** 新角色^[raw/articles/2026-04-29_深度访谈｜OpenClaw引爆Agent元年，AIAgent在企业内如何规模化应用？.md]

### 对 Agent 未来的判断

Agent 生态已进入百花齐放阶段。未来 Agent 会像水和电一样成为隐形基础设施，开源生态和端到端安全管控是重要引擎。^[raw/articles/2026-04-29_深度访谈｜OpenClaw引爆Agent元年，AIAgent在企业内如何规模化应用？.md]
