# Cursor

**类型**: AI 编码 IDE / Agent 平台
**创始人兼 CEO**: [[michael-truell]]
**定位**: AI Coding 领域领先的编辑器产品

## 概述

Cursor 是一款 AI 驱动的 [[ai-native-editor]]的代码编辑器，从 VS Code fork 而来，将 AI 能力深度嵌入编辑体验。2026年5月，创始人 Michael Truell 在 Cursor 新办公室（原《星球大战》首映影院）发表演讲《AI Coding 的下一个纪元》，提出了软件开发的三纪元模型。^[raw/articles/2026-05-14_Agent请求量暴涨15倍！Cursor创始人：75%企业代码已由AI生成，内部30%PR已由Agent端到端完成，探索无人类介入的自主Team模式.md]

## 三纪元模型

Michael Truell 将软件开发的发展划分为三个纪元：

| 纪元 | 名称 | 特征 |
|------|------|------|
| 1 | **Tab 时代** | 人类亲手编写每一行代码，用正式编程语言构建软件 |
| 2 | **Agent 时代**（当前） | 人们把编程任务交给 AI Agent，工程师变成 Agent 管理者 |
| 3 | **Team 时代**（未来） | 无人介入的自主 AI Agent 团队，端到端完成软件开发 |

## 关键数据（2026年5月）

- **Agent 请求量增长**：2025 年一年内，Agent requests 相较 Tab accepts 暴涨 15 倍
- **企业代码 AI 生成比例**：企业业务线约 **75%** 代码由 AI 生成（一年前仅 15-20%）
- **内部 PR 自动化**：Cursor 内部 **30%** 的 PR 由 Agent 端到端自主完成，无人类介入
- **Agent 自主实验**：Agent 团队一周内无人类参与开发出原型浏览器（300万行代码）

## Agent 管理者时代

工程师角色正在从"写代码"转向"管理 Agent"。Cursor 的数据显示：

- 人们越来越不直接触碰语法，转而把任务交给 Agents
- 工程师花更多时间在**审查**上——检查工作结果、查看构建版本、测试
- Agent 时代一次只能处理 1-3 个 Agent（共享代码库版本）
- Team 时代可以同时与几十个 Agent 协作（各自拥有独立电脑）

## 无人类介入实验

Cursor 做了一个实验：让 Agent 团队在完全无人类介入的情况下开发浏览器。一周内，Agent 团队生成了 300万行代码，从无法渲染 apple.com 进化到做出相当不错的网页渲染系统。目前仍处于早期实验阶段。^[raw/articles/2026-05-14_Agent请求量暴涨15倍！Cursor创始人：75%企业代码已由AI生成，内部30%PR已由Agent端到端完成，探索无人类介入的自主Team模式.md]
