# AI Agent & Skill 测评方案及落地实践

> **导语**：当 AI Agent 从"Demo 可用"走向"生产可靠"，测评就是那道必须跨过的门槛。

## 一、背景：为什么要做测评？

Agent 的自主性带来三个传统软件没有的问题：
1. **非确定性**：同一 prompt 多次执行结果不同
2. **黑盒化**：模型升级/Prompt微调可能导致行为漂移
3. **错误级联放大**：前序步骤的小偏差沿链路逐级放大

核心理念：**Eval = Agent 输入 → 执行 → 捕获执行过程（Trace + 产物） → 检查规则 → 可对比的分数**

## 二、测评框架

### 2.1 三类评委

| 评委 | 规则位置 | 成本 | 稳定性 | 覆盖能力 |
|------|---------|------|--------|---------|
| 确定性评分器（脚本） | 代码里 | 毫秒级/免费 | 100% | 已知硬指标 |
| Rubric评分器（LLM-as-Judge） | Prompt+JSON Schema | 秒级/API费 | 有抖动 | 未知软指标 |
| 人工评分器（专家） | 人脑+标注指南 | 分钟-小时级 | 取决于标注员 | 主观+边缘场景 |

**优先级**：确定性 > Rubric > 人工

### 2.2 五个测评维度

1. **功能正确性**（P0）— 结果正确性、任务完成度、指令遵循、工具调用正确性
2. **过程质量**（P1）— 推理合理性、步骤最优性、信息完整性、自我纠错能力
3. **效率与成本**（P1）— Token消耗、工具调用次数、端到端延迟、失败重试率
4. **鲁棒性与安全**（P0）— 一致性pass^k、异常恢复、抗对抗、幻觉率、越权风险
5. **体验与对齐**（P2）— 语气风格、回复清晰度、主动澄清、同理心

## 三、测评实施

### 3.1 用例集设计四大场景

**触发 → 核心逻辑 → 产物质量 → 异常容错**，每个场景含正向+负向用例。核心逻辑用例量最大（通常是其他三类之和的2-3倍）。

### 3.2 评分规则

100分扣分制：
- 任务结果不符：-100分
- 步骤不符：每步-10分
- 调用工具不符：每项-10分
- 超时（基准5分钟）：每多1分钟-10分
- 达标线：80分

### 3.3 基线管理

先跑一轮，人工确认过程和结果OK后保留为基线快照。后续执行对比基线评分。基线更新时机：Agent逻辑变更、模型升级、用例修改。

### 3.4 能力测评 vs 回归测评

- **能力测评**：起步20-50%通过率，指导优化方向，频繁迭代
- **回归测评**：接近100%，防御漂移，CI每次运行，只增不减

## 四、工程落地

### 4.1 前置依赖

被测Agent必须输出结构化Trace（JSONL/JSON格式），包含工具调用、思维链、中间产物。

### 4.2 环境隔离

Clone仓库到临时目录，每个用例执行前重置环境。

### 4.3 稳定性评估

多轮执行（N=5）检测非确定性：
- 关键决策类：0%容忍（全部通过）
- 辅助分析类：≤10%
- 创意生成类：≤40%

### 4.4 测评报告

包含全局概览、用例列表、单用例详情（多Trial聚合）、稳定性评分、逐Trial明细。

## 五、实战案例：TPerf性能AI分析Agent

- **Agent类型**：功能工具，部署在Knot平台，通过MCP调用TPerf-API
- **评分标准**：确定性评分（操作步骤+效率，最多-20分）+ 模型评分（关键判定+章节内容，最多-80分）
- **用例集**：9大场景分组、30+用例，覆盖CPU/网卡/磁盘/内存/TCP等瓶颈场景
- **模型对比**：7个模型并行评测，输出pass@1/pass^5/得分/Token/费用/耗时对比表
- **基线管理**：以Knot会话标识形式保存，评测时通过API动态获取
- **流水线**：基线确认流水线 + 正式测评流水线（pytest + TCase平台）
