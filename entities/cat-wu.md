# Cat Wu

**身份**: Anthropic Claude Code 和 Cowork 产品负责人

## 背景

工程师出身，职业路径：

1. **Scale AI** — 产品工程师
2. **Dagster Labs** — 工程经理
3. **Index Ventures** — 风投
4. **Anthropic** — Claude Code / Cowork 产品负责人

在 Anthropic 面试了几百个 PM 候选人，发现大多数仍以 6-12 个月路线图的思维找工作，与 Anthropic 一周甚至一天发布一个功能的节奏严重脱节。

## 核心观点

### AI Native PM

Cat Wu 认为传统 PM 的核心技能——做路线图、排期——在 AI 时代已被颠覆。AI Native PM 的核心是**缩短从想法到用户使用的周期**。具体要求：^[raw/articles/2026-04-25_ai_pm_catwu.md]

- 懂模型边界：知道当前模型能做什么、不能做什么
- 亲自试错：不要只看报告，要自己上手用
- 定义关键任务体验：聚焦用户完成特定任务的全流程

### "恰好正确程度的 AGI 信仰"

Cat Wu 认为 PM 最难的技能是保持**恰到好处的乐观**——

- **太乐观**：忽略当前用户的真实痛点，做出空中楼阁的功能
- **太保守**：在模型快速升级时措手不及，被竞争对手超越

### Team Principles 替代 PRD

在 Anthropic，Claude Code 团队用 **Team Principles** 替代常规 PRD。Team Principles 定义核心用户和取舍原则，让团队每个人都能自主决策，无需等待 PM 逐条确认。

### PM 与工程师角色融合

Cat Wu 认为 **PM 和工程师这两个角色在融合**——多招有产品品味的工程师，或多招 PM 来指导工程方向，效果差不多。Anthropic 更倾向于招有产品品味的工程师，传统 PM 不是第一选择。^[raw/articles/2026-04-27-anthropic-launch-secrets.md]

团队中有不少工程师可以从看到用户反馈开始，到周末就自己上线一个功能，几乎不需要 PM 参与。但融合的代价是**产品一致性**——有时同一个需求会有两个功能同时在做，让新用户困惑。

### "代码变便宜了，决定写什么变贵了"

> "代码变得越来越便宜了。那什么变得更有价值呢？决定写什么。"^[raw/articles/2026-04-27-anthropic-launch-secrets.md]

### Just do things

Cat Wu 的人生座右铭：**Just do things**——理解约束条件，想出该做什么，快速行动，从错误中学习。^[raw/articles/2026-04-27-anthropic-launch-secrets.md]

### 别停在 95%

关于自动化：一个自动化如果不能 100% 工作，就不是真正的自动化。最后那 5% 确实需要更多时间，但必须投入精力打磨到完全可靠。^[raw/articles/2026-04-27-anthropic-launch-secrets.md]

## 工作方式

- 花 **30% 时间**把 Cowork 推到极限（Cowork 是 Anthropic 的非代码 AI 产品，处理 Slack、deck、文档等）
- 与 [[boris-cherny]] 搭档，Boris 负责技术，Cat 负责产品
- 主导了 [[claude-code]] 功能交付周期从半年压缩到一天的转型
- 用 Cowork 为 Code with Claude 大会制作 20 页演讲稿（约 1 小时完成），连接 Google Calendar、Slack、Gmail、Google Drive，Cowork 自动翻阅 X 发布记录和内部频道整理内容^[raw/articles/2026-04-27-anthropic-launch-secrets.md]
- 产品使用策略：输出是代码的用 Claude Code，输出不是代码的（PPT、文档、邮件）用 Cowork^[raw/articles/2026-04-27-anthropic-launch-secrets.md]
